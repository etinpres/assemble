# shipper Step 4 — local tag + SHIP_REPORT render

You are dispatched as shipper Step 4 sub-agent. Print exactly one final line `WROTE: <abs path to SHIP_REPORT.md>` on stdout when done. No other prose. Main parses regex `^WROTE: (.+)$`; `server.harness.extract_wrote_paths` takes the **last match** (Spike VII F7).

## Inputs

- run_id: `{{RUN_ID}}`
- run_dir: `{{RUN_DIR}}`

Required files (read, never written):
- `{{RUN_DIR}}/preflight.json` — Step 1 output (verdict gate)
- `{{RUN_DIR}}/version_bump.json` — Step 2 output (may have `skipped: true`)
- `{{RUN_DIR}}/build_result.json` — Step 3 output (may have `skipped: true`)
- `{{RUN_DIR}}/parsed_scope.json` — for `tag_prefix` (defaults to `"v"`)

Templates (read from disk, NOT regenerated):
- `~/.claude/skills/assemble/bundled/shipper/templates/SHIP_REPORT.md.template`
- `~/.claude/skills/assemble/bundled/shipper/templates/SHIP_REPORT_ABORT.md.template`

## Bash tool access GRANTED

Scope: **`git tag -a <name> -m <message>`** (annotated, local-only) and **`git rev-parse <tag>`** (read SHA), via `server.git_helpers` ONLY — argv-list `subprocess.run` (T8 mitigation), unsafe-name fail-fast (T6).

Allowed helpers: `git_tag_exists` (T5 collision pre-check), `git_create_tag` (annotated, local), `git_tag_sha` (SHA capture).

**Forbidden** (defense in depth):

- `git tag -f` / `--force` — overwriting existing tags (T6, **high**). Helpers expose no force flag; do not construct your own subprocess to add one.
- `git push` ANY form — remote interaction (T7, **high**). Includes `git push`, `git push origin <tag>`, `git push --tags`, `git push --follow-tags`.
- Any remote/network op: `git fetch`, `git pull`, `git remote`, `git ls-remote`, `git merge`, `git clone`.
- `shell=True`, `os.system`, backtick interpolation, or any path that lets caller-controlled strings reach a shell.

On collision this step records `blocked (tag)` + renders abort template — does **not** retry with `-f`. Deleting/renaming the existing tag is user-owned.

## Goal

Read 4 prior-step JSONs + templates → branch abort vs happy → optionally create local annotated tag + capture SHA → compute final verdict → render `SHIP_REPORT.md` (or abort variant) via `str.replace` → write `tag_result.json` (audit) + `SHIP_REPORT.md` (final) → emit ONE trailing `WROTE:` line pointing at `SHIP_REPORT.md`.

Run from assemble repo root (harness CWD). `python3` + stdlib + `server.git_helpers` only.

## Steps

### Step 4.1 — Read prior-step artifacts

```python
run_dir = Path("{{RUN_DIR}}")
preflight = json.loads((run_dir / "preflight.json").read_text(encoding="utf-8"))
version_bump = json.loads((run_dir / "version_bump.json").read_text(encoding="utf-8"))
build_result = json.loads((run_dir / "build_result.json").read_text(encoding="utf-8"))
parsed_scope = json.loads((run_dir / "parsed_scope.json").read_text(encoding="utf-8"))

TIMESTAMP = datetime.now(timezone.utc).strftime("%Y-%m-%dT%H:%M:%SZ")
```

`FileNotFoundError` / `json.JSONDecodeError` → fall through to § Error handling (minimal abort report, still emit WROTE).

### Step 4.2 — Branch on path

```python
# Defaults — ALL must be set before Step 4.6 / 4.9 to prevent NameError on abort fall-through.
tag_name = None
tag_sha = None
tag_sha_short = ""
tag_message = ""
tag_collision = False
create_stderr_excerpt = ""
verdict_category = "ok"   # one of: ok / preflight / version / build / tag
verdict_reason = ""

abort_path = (
    preflight.get("verdict") != "pass"
    or version_bump.get("skipped") is True
    or build_result.get("skipped") is True
)
build_failed = (
    build_result.get("skipped") is False
    and build_result.get("exit_code") not in (0, None)
)
if build_failed:
    abort_path = True  # build-fail also routes to abort template; no tag created
```

`abort_path` covers all blocked categories (preflight / version-skipped / build-skipped / build-fail). When True: skip Steps 4.3–4.5; the defaults above guarantee Step 4.6 + Step 4.9 reads succeed without NameError. Step 4.6 still computes `verdict` + `verdict_reason` based on which condition triggered the abort.

### Step 4.3 — Compute tag name (happy path only)

```python
tag_prefix = parsed_scope.get("tag_prefix") or "v"
new_version = version_bump["new_version"]  # non-None on happy path
tag_name = f"{tag_prefix}{new_version}"
tag_message = f"Release {new_version}\n\nGenerated by assemble shipper ★ run {{RUN_ID}}"
```

### Step 4.4 — Tag collision pre-check (T5)

```python
cwd = Path.cwd()
tag_collision = git_tag_exists(cwd, tag_name)
if tag_collision:
    verdict_category = "tag"
    verdict_reason = f"git tag '{tag_name}' already exists; manually delete or bump version"
    tag_sha = None; tag_sha_short = None; create_stderr_excerpt = ""
    abort_path = True  # render abort; tag was NOT created. No -f retry.
```

### Step 4.5 — Create tag + capture SHA

```python
create_result = git_create_tag(cwd, tag_name, tag_message)
if not create_result["ok"]:
    create_stderr_excerpt = (create_result.get("stderr", "") or "")[:300]
    verdict_category = "tag"
    verdict_reason = f"git tag failed: {create_stderr_excerpt}"
    tag_sha = None; tag_sha_short = None
    abort_path = True
else:
    sha_result = git_tag_sha(cwd, tag_name)
    if not sha_result["ok"]:
        create_stderr_excerpt = (sha_result.get("stderr", "") or "")[:300]
        verdict_category = "tag"
        verdict_reason = f"git rev-parse <tag> failed: {create_stderr_excerpt}"
        tag_sha = None; tag_sha_short = None
        abort_path = True
    else:
        tag_sha = sha_result["stdout"].strip()
        tag_sha_short = tag_sha[:7]
        create_stderr_excerpt = ""
```

### Step 4.6 — Compute final verdict (canonical)

```python
verdict = "ship-ready" if (
    preflight.get("verdict") == "pass"
    and version_bump.get("new_version") is not None
    and (build_result.get("exit_code") == 0 or build_result.get("skipped") is True)
    and tag_sha is not None
) else "blocked"
```

`verdict_category` ∈ `{ship-ready, preflight, version, build, tag}` — pick the FIRST matching bucket (order matters):

- `ship-ready` → reason = `f"all 4 steps passed; tag {tag_name} at {tag_sha_short}"`
- `blocked (preflight)` → reason = `preflight.get("reason")` verbatim (e.g. `"fail (dirty tree): ..."`, `"fail (verify failed): ..."`, `"aborted by user override"`)
- `blocked (version)` → reason = `f"version bump failed: {version_bump.get('skip_reason','unknown')}"`
- `blocked (build)` → if `build_result.get("timed_out")`: `"build timed out (300s)"`; else `f"build exited {build_result.get('exit_code')}"`
- `blocked (tag)` → collision message OR `f"git tag failed: {create_stderr_excerpt}"` (already pre-set in 4.4/4.5)

### Step 4.7 — Build HANDOFF_COMMANDS dynamically

Detect project type by file presence in `Path.cwd()` (shipper does NOT run any of these — see SECURITY.md §Hand-off):

```python
lines = []
if not abort_path:
    branch = preflight.get("branch") or "main"
    lines.append(f"- Push commit + tag: `git push origin {branch} && git push origin {tag_name}`")
    if (cwd / "package.json").is_file():
        lines.append("- Publish to npm: `npm publish`")
    if (cwd / "pyproject.toml").is_file():
        lines.append("- Build + upload to PyPI: `python -m build && python -m twine upload dist/*`")
    if (cwd / "Cargo.toml").is_file():
        lines.append("- Publish to crates.io: `cargo publish`")
    if len(lines) == 1:  # only the git-push fallback line
        lines.append("- (no registry detected — manual upload of build artifact if applicable)")
else:
    if verdict_category == "preflight":
        lines.append("- Resolve pre-flight failure (clean tree / verify pass), then re-run shipper.")
    elif verdict_category == "version":
        lines.append("- Address version-bump error in `version_bump.json.errors`, then re-run from Step 2.")
    elif verdict_category == "build":
        lines.append("- Inspect `build_result.json` for stderr; fix build, then re-run shipper.")
    elif verdict_category == "tag":
        lines.append(f"- Resolve tag collision: `git tag -d {tag_name}` (if intentional rebuild) OR bump version, then re-run.")
    else:
        lines.append("- Inspect run_dir artifacts; fix root cause; re-run shipper.")
handoff_commands = "\n".join(lines)
```

```python
template_root = Path.home() / ".claude/skills/assemble/bundled/shipper/templates"
if abort_path:
```

Build the shared (abort + happy) placeholder map first, then extend with happy-only keys when applicable. `{{RUN_ID}}` is **excluded** — left for upstream orchestrator substitution.

```python
files_changed = version_bump.get("files_changed") or []

# Spike IX cleanup CF-1 — abort detail computed per verdict_category so the

if abort_path:
    if verdict_category == "preflight":

    elif verdict_category == "tag":
        abort_detail = "Version bumped + build passed. Tag step blocked (see §Verdict reasoning). No tag created."
        abort_phase_note = "Tag step blocked — Sections 3 + 4 ARE in audit JSONs; Section 5 (Tag) was attempted but failed."
    else:
        abort_detail = "Pipeline aborted. See §Verdict reasoning + audit JSONs for partial completion details."
        abort_phase_note = "Sections 3–5 may be partially populated — check audit JSONs."
else:

# 10 shared placeholders (used by both happy and abort templates; 2 abort-only)
subst = {

    "{{CLEAN_TREE}}":       str(preflight.get("clean_tree", False)).lower(),
    "{{BRANCH}}":           preflight.get("branch") or "(unknown)",
    "{{HEAD_SHA}}":         preflight.get("head_sha") or "(unknown)",
    "{{VERIFY_VERDICT}}":   verify_field,
    "{{VERDICT_REASON}}":   verdict_reason,
    "{{HANDOFF_COMMANDS}}": handoff_commands,
    "{{ABORT_DETAIL}}":     abort_detail,

}

# 15 happy-only placeholders (skipped on abort path — abort template has only the 8 above + {{RUN_ID}})
if not abort_path:
    subst.update({
        "{{TAG_NAME}}":          tag_name or "(none)",
        "{{TAG_SHA}}":           tag_sha or "(none)",
        "{{TAG_SHA_SHORT}}":     tag_sha_short or "(none)",
        "{{NEW_VERSION}}":       str(version_bump.get("new_version") or "(none)"),
        "{{OLD_VERSION}}":       str(version_bump.get("old_version") or "(none)"),
        "{{RELEASE_KIND}}":      str(version_bump.get("release_kind") or "(none)"),
        "{{VERSION_FORMAT}}":    str(version_bump.get("version_format") or "(none)"),
        "{{FILES_CHANGED}}":     ", ".join(f"`{f}`" for f in files_changed) if files_changed else "(none)",
        "{{CHANGELOG_STATUS}}":  str(version_bump.get("changelog_status") or "(none)"),
        "{{BUILD_COMMAND}}":     str(build_result.get("build_command") or "(skipped)"),
        "{{BUILD_EXIT_CODE}}":   str(build_result.get("exit_code")),
        "{{BUILD_DURATION_MS}}": str(build_result.get("duration_ms", 0)),
        "{{BUILD_TRUNCATED}}":   str(build_result.get("truncated", False)).lower(),
        "{{BUILD_TIMED_OUT}}":   str(build_result.get("timed_out", False)).lower(),
        "{{TAG_MESSAGE}}":       "`" + (tag_message.splitlines()[0] if tag_name else "(none)") + "`",
    })

body = template_text
```

### Step 4.10 — Emit WROTE (final)


The audit `tag_result.json` is intentionally NOT echoed via a `WROTE:` line — Step 4's canonical artifact is the rendered report. Spike VII F7 last-match semantics permit multiple `WROTE:` lines, but this step deliberately emits only one.

## Output discipline

Single trailing `WROTE: <abs path to SHIP_REPORT.md>` line. Orchestrator parses regex `^WROTE: (.+)$` and takes the **last** match (Spike VII F7). Do NOT print prose, banners, progress dots, or warnings on stdout. Diagnostics belong inside the report's §Verdict reasoning section or `tag_result.json.stderr_excerpt`.

## Error handling

Any unhandled exception during Steps 4.1–4.9 → write a minimal `SHIP_REPORT_ABORT.md` with:

- `verdict = "blocked"`
- `verdict_reason = f"step 4 unhandled error: {type(exc).__name__}: {exc}"`
- `handoff_commands = "- Inspect run_dir artifacts and run logs; file as a shipper bug if reproducible."`
- placeholders `clean_tree` / `branch` / `head_sha` / `verify_verdict` filled with `"(unknown)"` if preflight could not be read.

Still emit `WROTE: <SHIP_REPORT.md path>` so the orchestrator can advance the dispatch chain. Exit 0 — the orchestrator detects via the report's verdict, not via exit code.
