## Sub-agent dispatch pattern (per task)

Per `superpowers:subagent-driven-development`, each task above is implemented by:

1. **Implementer sub-agent** — `general-purpose` (or `codex:codex-rescue` for E1) — receives task spec + acceptance criteria + dependency context. Returns code changes.
2. **Spec-review sub-agent** — `general-purpose` — receives implementer's diff + original task spec. Returns PASS/FAIL with explicit gap list.
3. **Code-quality review sub-agent** — `superpowers:code-reviewer` — receives diff + plan reference. Returns review notes.
4. **Main Claude (orchestrator)** — receives all 3 reports, applies fixes if needed, commits as single atomic commit.

Phase E task E1 uses `codex:codex-rescue` directly per its skill description.

| Gate | Trigger | Action |
|---|---|---|
| Phase E mandatory | All Phase A–D tasks landed | Block Phase F until E3 done |
| Bash surface audit | Any Bash-granted prompt body change | Re-trigger E1 (Codex retro) |
| Tag collision | `git_tag_exists` returns true during F2 dogfood | Use `vSPIKE-IX-DOGFOOD-{timestamp}` tag prefix in dogfood instead of normal `v0.X.Y` |

| Pytest regression | Any task drops baseline below 449 | Block ship; root-cause + fix before next task |
