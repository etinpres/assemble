### Implementation notes (carry into A1 prompt)

- Use `shutil.copytree(src, tmp_inner, copy_function=shutil.copy2)` for the actual copy.
- Generate temp dir name as `f"{dest.name}.tmp.{os.getpid()}.{secrets.token_hex(4)}"` parented at `dest.parent` (so atomic rename within same filesystem).
- Backup name on overwrite: `f"{dest.name}.bak.{int(time.time())}"`.
- `validate_dest_name` regex: `^[a-z][a-z0-9_-]{0,63}$`. Reject reserved names BEFORE regex (clearer error message).
- `EjectError` subclasses `Exception` (not `ValueError`) so callers can `try: ... except EjectError as e: print(e)` without colliding with stdlib value-errors.
- `dry_run_plan` walks `src.rglob("*")` filtering files only (not dirs), accumulates total_bytes via `f.stat().st_size`.

### Anti-patterns (deny list)

- ❌ Mutating `src` in any code path (eject is read-only on source).
- ❌ Calling `subprocess.run` or `os.system` (pure shutil/pathlib only).
- ❌ Path traversal beyond `~/.claude/skills/<name>/` for dest.
- ❌ Silent overwrite without backup (always create `.bak.<ts>` when overwrite=True).
- ❌ Auto-cleanup of `.bak.<ts>` survivors (user must remove manually).
- ❌ Frontmatter rewriting / SKILL.md text mutation (eject is byte-faithful copy).
- ❌ Adding to `_BUNDLES` or `ALLOWED_PROMPT_FILES` or any harness/inventory state.

### Commit message template

- Backup-on-overwrite (.bak.<ts>, no auto-cleanup)
