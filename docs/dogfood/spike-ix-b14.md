## Setup

Self-execute orchestrator at `/tmp/spike-ix-b14/run_dogfood.py` simulates each Step 1→4 by directly invoking the production helpers the prompt bodies call. No `Bash` calls from main during the dispatch chain — every git probe goes through `server.git_helpers` (argv-list only), version arithmetic through `server.version_helpers`, scope parsing through `server.scope_parser`, template substitution via `str.replace` over the bundled templates.

- **Happy**: 4 (Step 1, 2, 3, 4) — matches prediction.
- **Abort**: **2** (Step 1, then Step 4 abort render — Steps 2 + 3 skipped per spec § Step 4 fail-path: "fail aborts dispatch chain — Step 2/3/4 not dispatched. Final SHIP_REPORT still rendered in Step 4-fail-path"). Plan predicted 4. Discrepancy: plan optimistically assumed all 4 dispatches still fire for audit-trail symmetry; spec actually skips 2 + 3.
- **Build-fail**: 4 (Step 1, 2, 3 all run; Step 4 dispatched and renders abort template). Plan predicted 3 (Step 4 skipped). Discrepancy: spec § Step 4 build-fail branch *does* dispatch Step 4 (renders ABORT template, no tag created). Both 3 and 4 are defensible; current self-execute = 4.

Recommendation: **Spike X amendment** to spec/plan AC8 — abort=2, buildfail=4 (matching spec semantics).
